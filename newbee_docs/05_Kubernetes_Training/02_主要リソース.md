# 02. 主要リソースの理解 🧱

K8sを扱う上で避けて通れない「4つの基本オブジェクト」を覚えましょう。

## 1. Pod (ポッド) 🥚
- **K8sにおける最小単位**です。
- 1つ以上のコンテナ（Dockerコンテナ）が入っています。
- 通常は「Pod ≒ コンテナ1つ」と考えてOKですが、密接に関係するサブコンテナをセットにすることもあります。
- **注意**: Podは使い捨てです。死んだら生き返らず、新しいPodが代わりに作られます。

## 2. ReplicaSet (レプリカセット) 👯
- 「Podのコピー（レプリカ）を常に○個維持する」役割を持ちます。
- 「WebサーバーのPodを3つ」と指定すれば、常に3つあるように監視してくれます。

## 3. Deployment (デプロイメント) 🚀
- ReplicaSetの親分です。実務ではこれを作ります。
- 「アプリのバージョンアップ」を管理します。
- 例：「v1.0のPod 3つ」から「v2.0のPod 3つ」へ、**サービスを止めずに順番に入れ替える（ローリングアップデート）** ことができます。

## 4. Service (サービス) 💁
- Podへの「窓口（受付）」です。
- Podは頻繁に作られたり消えたりしてIPアドレスが変わるため、外部から直接通信するのは大変です。
- Serviceは固定のIPアドレスを持ち、届いた通信を裏側にいる複数のPodに**振り分け（ロードバランス）** してくれます。

## 関係性のまとめ
1. **Deployment** に「アプリv1を3つ動かしたい」と書く。
2. Deploymentが **ReplicaSet** を作り、ReplicaSetが **Pod** を3つ作る。
3. ユーザーは **Service** にアクセスし、Serviceが3つの **Pod** のどれかに繋ぐ。

---

## 📚 用語集 (Glossary)

| 用語 | 意味 | イメージ |
|------|------|-----|
| **Pod** | ポッド | 豆のさや。コンテナの入れ物。最小単位。 |
| **ReplicaSet** | レプリカ管理 | 「常に3つ」を維持する監視員。 |
| **Deployment** | 展開・配備 | バージョンアップを取り仕切る現場監督。 |
| **Service** | サービス | 窓口・受付係。 |
| **Rolling Update** | ローリングアップデート | 止めずに少しずつ入れ替える更新方法。 |
| **Load Balancing** | 負荷分散 | アクセスを均等に振り分けること。 |
