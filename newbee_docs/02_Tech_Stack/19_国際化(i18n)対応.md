# 06_国際化(i18n)への対応 🌏

TASUKIが将来、海外展開するかもしれません。
コードの中に `'設定'` とか `'Save'` といった文字列を直接書く（ハードコードする）と、後で地獄を見ます。

## 1. l10n と i18n
- **i18n (Internationalization - 国際化)**: アプリを多言語に対応**できる仕組み**にすること。エンジニアの仕事。
- **l10n (Localization - 地域化)**: 実際に翻訳データを入れ、通貨や日付フォーマットをその国に合わせること。翻訳者の仕事。

## 2. arbファイル (Application Resource Bundle)
Flutter の標準的な翻訳ファイル形式です（JSONに似ています）。

`app_en.arb`:
```json
{
  "hello": "Hello, {name}!",
  "@hello": {
    "description": "Greeting message",
    "placeholders": {
        "name": {}
    }
  }
}
```

`app_ja.arb`:
```json
{
  "hello": "こんにちは、{name}さん！"
}
```

## 3. コードからの利用
`flutter_localizations` パッケージを使うと、arbファイルから自動的に Dart クラスを生成してくれます。

```dart
// 直接書かない！
// Text("こんにちは、Tanakaさん！")

// 自動生成コードを使う
Text(AppLocalizations.of(context)!.hello("Tanaka"))
```
これなら、スマホの言語設定を変えるだけで、自動的に日本語/英語が切り替わります。

## 4. RTL (Right-to-Left) 対応
アラビア語やヘブライ語は「右から左」に読みます。
Flutter は優秀なので、`Start` (左) / `End` (右) という概念でレイアウトしていれば、RTL言語環境では自動的にレイアウトを鏡写しに反転してくれます。
`Left` / `Right` と指定するより `Directional` な指定（Leading / Trailing）を使う癖をつけましょう。

## 📚 用語集 (Glossary)

| 用語 | 意味 | 例 |
|------|------|-----|
| **i18n** | internationalization | iとnの間に18文字あるから |
| **l10n** | localization | lとnの間に10文字あるから |
| **arb** | Application Resource Bundle | Flutter用の翻訳ファイル形式 |
| **RTL** | Right To Left | 右書き言語 (アラビア語など) |
| **Locale** | 言語と地域の組み合わせ | `ja_JP` (日本語・日本), `en_US` (英語・米国) |

## 🛠️ コマンド集 (i18n)

| コマンド | 説明 |
|---|---|
| `flutter gen-l10n` | arbファイルからDartコードを生成する |
