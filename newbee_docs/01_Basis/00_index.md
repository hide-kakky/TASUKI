# 01_Basis 学習ガイド 🧭

このフォルダのゴール: Web/アプリ開発の土台となる用語と仕組みを一気に俯瞰する。

## 推奨順とミッション
- 所要時間の目安: 1本5〜10分。ミッションは2〜5分。
- 前提: プログラミング未経験を想定。ターミナルが動く環境。
- 01_プログラミングとは（★）  
  - ミッション: 「プログラムが現実を変える例」を身近な体験で1つ書く。
- 02_TASUKIアプリの全体像（★）  
  - ミッション: TASUKIの「入力→処理→出力」を3行で要約。
- 03_開発環境の理解（★）  
  - ミッション: 使っているエディタとターミナルを確認し、1つショートカットを調べて使う。
- 04_データベースとは（★）  
  - ミッション: テーブル/レコード/カラムをそれぞれ1行で説明。
- 05_フロントエンドとバックエンド（★）  
  - ミッション: フロント/バックがそれぞれ「誰に何を返すか」を1文ずつ。
- 06_APIとは（★）  
  - ミッション: GET/POST/PATCH/DELETEでプロフィールを扱うURL例を書き出す。
- 07_HTTPとREST_API（★）  
  - ミッション: ステータスコード200/401/403/404の意味を1行ずつ。
- 08_JSONとデータ構造（★）  
  - ミッション: 自分のプロフィールをJSONで表現し、配列とオブジェクトの違いを書く。
- 09_認証と認可の仕組み（★★）  
  - ミッション: 認証と認可の違いを10秒で説明する台本を作る。
- 10_ターミナルとシェル操作入門（★）  
  - ミッション: `pwd`, `ls`, `cd`, `cat`を実行し、結果を1行ずつメモ。
- 11_コードの読み方（★）  
  - ミッション: 任意の短い関数を読み、入出力と副作用を箇条書きにする。
- 12_OSとプロセスについて（★★）  
  - ミッション: プロセスとスレッドの違いを例え話で1文にする。
- 13_ネットワークとセキュリティ（★★）  
  - ミッション: TLSハンドシェイクを3ステップで書く。
- 14_アルゴリズムとデータ構造（★★）  
  - ミッション: 時間計算量O(1)/O(n)/O(log n)の例を1つずつ挙げる。

## 進捗チェック
- [ ] 01〜07 読了＆各ミッション1つ以上実施
- [ ] 08〜11 読了＆ターミナル/コード読み練習済み
- [ ] 12〜14 読了＆計算量/ネットワークを自分の言葉で説明できる

## 速習のコツ
- 1本5〜10分、ミッションを必ず1つやってから次へ進む。
- 分からない用語は`glossary.md`と照らし合わせる。
