# 10_ターミナルとシェル操作入門 💻

黒い画面（ターミナル）は怖くありません。
GUI（マウス操作）よりも圧倒的に速く、正確にコンピュータに命令するための道具です。

## 1. これだけは覚えよう！基本コマンド

| コマンド | 意味 | 覚え方 | 例 |
|---|---|---|---|
| `ls` | ファイル一覧を表示 | **L**i**s**t | `ls -la` (詳細＆隠しファイルも) |
| `cd` | ディレクトリ移動 | **C**hange **D**irectory | `cd apps/app` |
| `pwd` | 今どこにいるか表示 | **P**rint **W**orking **D**irectory | `pwd` |
| `mkdir` | フォルダ作成 | **M**a**k**e **Dir**ectory | `mkdir new_folder` |
| `rm` | 削除（取り消せないので注意！） | **R**e**m**ove | `rm file.txt` |
| `cp` | コピー | **C**o**p**y | `cp original.txt copy.txt` |
| `mv` | 移動・名前変更 | **M**o**v**e | `mv old.txt new.txt` |

## 2. パス (Path) の概念
ファイルの住所のことです。

- **絶対パス**: 最上階（ルート `/`）からの住所。
  - 例: `/Users/hide_kakky/Dev/TASUKI/README.md`
  - どこにいても必ずたどり着ける。
- **相対パス**: 今いる場所（カレントディレクトリ）からの道順。
  - `.` : 今ここ
  - `..` : 一つ上の階
  - 例: `../README.md` (隣の部屋のファイル)

## 3. 便利なショートカット
- **Tabキー**: 入力補完。`cd app` まで打って Tab を押すと `cd apps/` になったりします。必須です。
- **↑ / ↓キー**: 履歴呼び出し。さっき打った長いコマンドをもう一度呼び出せます。
- **Ctrl + C**: 強制終了。「なんか止まらない！」「間違えた！」と思ったらこれで脱出。

## 4. パイプ `|` と リダイレクト `>`
- **パイプ `|`**: コマンドの結果を次のコマンドに渡す。
  - `ls | grep .md` （ファイル一覧の中から `.md` を含むものだけ探す）
- **リダイレクト `>`**: 結果をファイルに保存する。
  - `echo "Hello" > hello.txt` （Helloと書いたファイルを作る）

## 5. 環境変数 (`export`)
APIキーなどの秘密情報は、コードに書かずに環境変数に入れます。
```bash
export API_KEY="secret_123"
echo $API_KEY
```
TASUKIでは `.env` ファイルに書いておくと、ツールが勝手に読み込んでくれます。

## 📚 用語集 (Glossary)

| 用語 | 意味 | 例 |
|------|------|-----|
| **ターミナル** | コマンドを入力する黒い画面のアプリ | VSCodeのターミナル, iTerm2 |
| **シェル (Shell)** | コマンドを解釈して実行するプログラム | zsh, bash |
| **パス (Path)** | ファイルやフォルダの場所（住所） | `/Users/name/Docs` |
| **カレントディレクトリ** | 今いる場所 | `.` (ドット) |
| **ホームディレクトリ** | ユーザーの拠点 | `~` (チルダ) |
| **ルートディレクトリ** | 一番上の階層 | `/` (スラッシュ) |
| **コマンド** | コンピュータへの命令 | `ls`, `cd` |
| **引数 (Arguments)** | コマンドに渡すオプション情報 | `ls -a` の `-a` |
