# 03_Process 学習ガイド 🧭

ゴール: TASUKIでの開発フロー、テスト戦略、運用の基本を押さえる。

## 推奨順とミッション
- 所要時間の目安: 1本10分。テンプレ記入に+5分。
- 前提: Basisを終え、簡単なGit操作ができる状態。
- 01_実装の流れ（★）  
  - ミッション: 「設計→実装→テスト→デプロイ」を自分の作業に当てはめて3行で書く。
- 02_テストと環境分離（★）  
  - ミッション: ローカル/ステージング/本番の違いを表にして1行ずつ用途を書く。
- 03_コードレビューの歩き方（★）  
  - ミッション: レビューコメント例を「観点＋理由＋提案」の型で1つ書く。
- 04_良いIssueの書き方（★）  
  - ミッション: タイトル/概要/再現手順/期待結果/スクショのテンプレを自分用に書く。
- 05_デバッグの基本（★）  
  - ミッション: 「再現条件」「期待と実際」「仮説」の3点セットで短いメモを作る。
- 06_テスト戦略完全ガイド（★★）  
  - ミッション: ユニット/UI/E2Eの役割を1行ずつ。
- 07_アーキテクチャと依存注入（★★★）  
  - ミッション: 依存注入で得られる利点を2つ書く。
- 08_CI_CDパイプライン構築（★★）  
  - ミッション: パイプラインのステージ（Lint/Unit/UI/E2E/Deploy）を順番に並べる。
- 09_SREと可観測性（★★★）  
  - ミッション: SLOとSLIの違いを1行で説明。

## 進捗チェック
- [ ] 01〜04 読了＆自分用PR/Issueテンプレを作成
- [ ] 05〜06 読了＆バグ切り分けメモ/テスト観点を記述
- [ ] 07〜09 読了＆CI/CDステージとSLO/SLIを自分の言葉で説明

## 速習のコツ
- 各章のミッションを、自分の進行中タスクにそのまま貼り付けて練習する。
- 03と04はPR/Issueを書く前に必ず見返す。
