# 06_Git_Github_Training 学習ガイド 🧭

ゴール: Git/GitHubでチーム開発を安全に進める型を身につける。

## 推奨順とミッション
所要時間の目安: 1本10〜15分（手元で演習する場合+10分）。  
前提: Basisを終え、簡単なターミナル操作ができること。
1. 01_Gitの基本概念（★）  
   - ミッション: コミット・ブランチ・マージを1文ずつ説明。
2. 02_基本コマンドとフロー（★）  
   - ミッション: `add -> commit -> push` の流れを図にする。
3. 03_GitHubとチーム開発（★）  
   - ミッション: PRに含めるべき情報（概要/スクショ/確認項目）をテンプレ化。
4. 04_ブランチ戦略（★★）  
   - ミッション: GitFlowとTrunk-Basedの違いを1文ずつ。
5. 05_リベースとCI_CD（★★★）  
   - ミッション: Rebaseの利点と注意点を2行で書く。
6. 06_Git内部構造入門（★★★）  
   - ミッション: コミットオブジェクトとツリーオブジェクトの役割を1行ずつ。

## 進捗チェック
- [ ] 01〜02 実行＆手元でadd/commit/pushを試す
- [ ] 03〜04 読了＆PRテンプレとブランチ戦略を決定
- [ ] 05〜06 読了＆rebase演習・内部構造の理解メモを作成

## 速習のコツ
- 小さなリポジトリを作って、PR作成→レビュー→マージの流れを自演する。
- 競合を恐れず、手元で`rebase`と`merge`を試して挙動を確認する。
