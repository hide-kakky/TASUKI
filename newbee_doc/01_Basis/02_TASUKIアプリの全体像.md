## 1. TASUKIとは何か？

### ひとことで言うと

**「飲食店の暗黙知を、動画とAIで形式知化するアプリ」**

...難しいですね😅

もっと簡単に言うと：

**「ベテランの知識を新人に簡単に伝えるアプリ」**

### 課題：飲食店の「暗黙知」問題

#### 暗黙知とは？
「頭の中にはあるけど、言葉にしにくい知識」のこと。

**例：オーダーの取り方**
- ❌ マニュアル: 「お客様から注文を聞く」
- ✅ ベテラン: 「笑顔で近づいて、まずドリンクを聞いて、メニュー番号で復唱して...」

ベテランは**経験で身につけた細かいコツ**を知っていますが、新人に全部教えるのは大変です。

#### 結果：新人が辞めてしまう
1. 教わることが多すぎて覚えられない
2. 「前の人に聞いて」と言われるけど、誰に何を聞けばいいかわからない
3. ミスして怒られる
4. 辞めてしまう😢

**→ これを解決するのがTASUKI！**

---

## 2. TASUKIの解決方法

### Flow（フロー）→ Stock（ストック）の仕組み

```
【従来】
ベテランの知識 → 口頭で伝える → すぐ忘れる ❌

【TASUKI】
ベテランの知識 → 動画で記録 → AIが整理 → いつでも見返せる ✅
```

#### Flow（流れ）= 動画を撮る
スタッフが業務を動画で撮影
- 「オーダーの取り方」
- 「料理の盛り付け方」
- 「レジの締め方」

#### Stock（蓄積）= マニュアル化
AIが動画を分析して、自動でマニュアル作成
- 要約
- 手順（ステップバイステップ）
- コツ・注意点

## 3. 主な機能（3つのペルソナ）

TASUKIには3種類のユーザーがいます：

### 👔 Owner（オーナー）
**役割**: 店舗全体を管理する人

**できること**:
- 店舗の作成・削除
- Managerの任命
- 全マニュアルの閲覧

### 👨‍💼 Manager（店長）
**役割**: マニュアルを管理する人

**できること**:
- **AIが作ったマニュアルを承認する** ← これが一番重要！
- スタッフの招待
- マニュアルの編集
- Google Docsからマニュアルを取り込む

### 👥 Staff（スタッフ）
**役割**: 現場で働く人

**できること**:
- **動画を撮影する** ← Flow作成
- 公開されたマニュアルを見る
- カテゴリで検索する

## 4. 使用シーン（実際の流れ）

### シーン1: 新しいマニュアルを作る

```
1. 【Staff】ベテランスタッフが「オーダーの取り方」を実演しながら動画撮影
   ↓
2. 【自動】動画がサーバーにアップロードされる
   ↓
3. 【AI】Geminiが動画を分析
   - 「お客様からオーダーを伺い、ハンディに入力する」（要約）
   - Step 1: テーブルに近づく
   - Step 2: 「お決まりですか？」と声をかける
   - ...
   ↓
4. 【Manager】店長が内容を確認して承認
   ↓
5. 【自動】全スタッフが見られるように公開される
```

### シーン2: 新人がマニュアルを見る

```
1. 【Staff】新人スタッフがアプリを開く
   ↓
2. 「ホール」カテゴリを選択
   ↓
3. 「オーダーの取り方」をタップ
   ↓
4. 動画と手順を見ながら学習
   ↓
5. わからなかったら何度でも見返せる ✅
```

## 5. システム全体の構成

TASUKIは以下の4つの部分で構成されています：

```
┌────────────────────────────────────────┐
│  📱 Flutter (スマホアプリ)              │
│  - 動画撮影                             │
│  - マニュアル閲覧                       │
│  - タイムライン表示                     │
└────────────────────────────────────────┘
         ↕ (通信)
┌────────────────────────────────────────┐
│  🔧 Supabase (バックエンド)             │
│  - ユーザー認証                         │
│  - データ保存                           │
│  - 権限管理（RLS）                      │
└────────────────────────────────────────┘
         ↕
┌────────────────────────────────────────┐
│  🎬 Mux (動画処理サービス)              │
│  - 動画のアップロード                   │
│  - 動画の変換（HLS形式）                │
│  - 動画の配信                           │
└────────────────────────────────────────┘
         ↕
┌────────────────────────────────────────┐
│  🤖 Gemini (AI)                         │
│  - 動画の分析                           │
│  - マニュアルの自動生成                 │
│  - 多言語対応                           │
└────────────────────────────────────────┘
```

**それぞれの役割**:
- **Flutter**: スタッフが見る画面
- **Supabase**: データを保存・管理
- **Mux**: 動画を処理・配信
- **Gemini**: AIでマニュアル作成

## 6. 重要な専門用語（覚えよう！）

| 用語 | 意味 | TASUKIでの使われ方 |
|------|------|-------------------|
| **Flow（フロー）** | 流れ、動的なもの | スタッフが撮影する動画 |
| **Stock（ストック）** | 蓄積、静的なもの | AIが作ったマニュアル |
| **暗黙知** | 言葉にしにくい知識 | ベテランのコツ |
| **形式知** | 文書化された知識 | マニュアル |
| **ペルソナ** | 典型的なユーザー像 | Owner / Manager / Staff |
| **カテゴリ** | 分類 | ホール/キッチン/清掃等 |
