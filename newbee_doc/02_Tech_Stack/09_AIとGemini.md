## 1. TASUKIにおけるAIの役割

しかし、最新のAIなら可能です！

---

## 2. Gemini（ジェミニ）とは？

Googleが開発した最新のAIモデルです。

### マルチモーダル（Multimodal）

**Multi（複数の） + Modal（モード・形式）**

テキストだけでなく、**画像、音声、動画** を理解できるAIのことです。

```
【従来のAI】
テキスト入力「カレーの作り方を教えて」
  ↓
テキスト出力「1. 玉ねぎを切ります...」

【Gemini】
動画入力（カレーを作っている動画）
  ↓
テキスト出力「動画では、まず玉ねぎを薄切りにしていますね。次に...」
```

**TASUKIにGeminiが最適な理由**:
動画を直接アップロードして、「この動画の手順を書き出して」と頼めるからです！

---

## 3. プロンプトエンジニアリング

AIへの指示の出し方（プロンプト）を工夫することです。

### TASUKIのプロンプト例

```text
あなたはプロの飲食店マニュアル作成者です。
この動画を見て、以下のJSON形式で出力してください。

{
  "summary": "動画の要約（50文字以内）",
  "steps": [
    {
      "step_number": 1,
      "description": "具体的な行動",
      "tips": "コツや注意点"
    }
  ]
}

注意点：
- 初心者にもわかりやすい言葉を使ってください
- 衛生管理に関する動作があれば強調してください
```

**工夫ポイント**:
1. **役割を与える**（プロの作成者）
2. **出力形式を指定する**（JSON）
3. **制約条件をつける**（50文字以内、初心者向け）

---

## 4. AIの限界と人間の役割

AIは完璧ではありません。

- **ハルシネーション（嘘）**: 動画にないことを勝手に作り出すことがある
- **文脈の誤解**: 「塩」と「砂糖」を見間違えるかも

だからこそ、**Manager（店長）の承認** が必要なのです。

```
AIが下書きを作る（80%の完成度）
  ↓
人間が修正・承認する（100%にする）
```

これがTASUKIの設計思想です。
