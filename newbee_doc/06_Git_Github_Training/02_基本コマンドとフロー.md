# 02. 基本コマンドとフロー 🛤️

個人のPCで行う基本的な作業フローを覚えましょう。

## 基本の3ステップ
Gitの日常作業は、主にこの3つの繰り返しです。

1. **add**: 変更したファイルを「コミットする候補」としてステージ（演壇）に上げる。
2. **commit**: ステージ上の変更を「確定」してセーブする。
3. **push**: 手元のセーブデータをGitHub（リモート）に送信する。

## コマンド解説

### 1. 状態確認
```bash
git status
```
- 今どのファイルが変更されているか、どのファイルがアド（add）されているかを確認します。迷ったらまずはこれ！

### 2. ステージング (add)
```bash
git add リファイル名
# または、全ての変更を一括でadd
git add .
```
- ファイルの変更を「次回のコミットに含める」状態にします。

### 3. コミット (commit)
```bash
git commit -m "変更内容のメッセージ"
```
- 変更をローカルリポジトリに記録します。
- メッセージは重要です。「修正」だけでなく「〇〇機能の追加」「△△バグの修正」など具体的に書きましょう。

### 4. プッシュ (push)
```bash
git push
```
- 自分のPCにあるコミット履歴を、GitHub（リモートリポジトリ）にアップロードします。

### 5. プル (pull)
```bash
git pull
```
- GitHubにある最新の変更履歴を、自分のPCにダウンロードして取り込みます。
- チーム開発で他の人が変更した内容を手元に反映させるときに使います。

## ログ確認
```bash
git log
# 見やすく表示
git log --oneline --graph
```
- 過去のコミット履歴を確認できます。
