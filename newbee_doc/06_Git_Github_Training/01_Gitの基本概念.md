# 01. Gitの基本概念 🌳

## 1. Git（ギット）とは？
**「ファイルの変更履歴を保存・管理するツール（バージョン管理システム）」** です。
ゲームの「セーブデータ」を好きなタイミングで作れて、いつでも過去のセーブポイントに戻れるようなものです。

### 解決する問題
- 「最新版_v2_final_修正版.txt」のようなファイル地獄を防ぐ。
- 「昨日動いていたのになぜか動かなくなった！昨日の状態に戻したい！」を瞬時に実現する。

## 2. GitとGitHubの違い
よく混同されますが、全く別物です。

| 名前 | 役割 | 置き場所 |
| :--- | :--- | :--- |
| **Git** | 変更履歴を記録する**ツール** | あなたのパソコン (ローカル) |
| **GitHub** | 履歴データを保存・共有する**Webサービス** | インターネット上 (リモート) |

- **Git**: ノート（変更履歴）を書くためのペンと紙。
- **GitHub**: ノートのコピーを保管してみんなで見られる図書館。

## 3. 重要なキーワード

### 🗂️ リポジトリ (Repository)
- ファイルや変更履歴が保存されている「保管庫」のこと。ディレクトリ（フォルダ）単位で管理します。
- **ローカルリポジトリ**: 自分のPC内にある保管庫。
- **リモートリポジトリ**: GitHubなどのサーバー上にある保管庫。

### 📸 コミット (Commit)
- ある時点でのファイルの「スナップショット（セーブデータ）」を記録すること。
- コミットには必ず「何を変更したか」のメッセージを付けます。

### 🌿 ブランチ (Branch)
- 履歴の流れを分岐させる機能。
- 例：「本番用の安定版」から「新機能開発用の作業版」を枝分かれさせ、完成したら合流（マージ）させる。

## 📚 用語集 (Glossary)

| 用語 | 意味 | イメージ |
|------|------|-----|
| **VCS** | Version Control System | バージョン管理システム (=Git) |
| **Repository (リポジトリ/Repo)** | 貯蔵庫 | 全ての履歴が入っている箱 |
| **Local** | ローカル | あなたの手元のPC |
| **Remote** | リモート | ネットの向こう側 (GitHubなど) |
| **Commit (コミット)** | 記録 | 歴史の1ページを作る |
| **Branch (ブランチ)** | 枝 | 平行世界を作る |
| **Merge (マージ)** | 統合 | 平行世界を一つにする |
| **Checkout (チェックアウト)** | 移動 | 別のブランチや過去へワープする |
