# 01. Kubernetes (K8s) の基本概念 ☸️

## 1. Kubernetes（クーバネティス/ケーエイツ）とは？
Dockerコンテナの**軍団を指揮・管理するためのツール**（コンテナオーケストレーションツール）です。

### 🚢 「港の管制塔」で例えると...
- **Docker**: コンテナ（荷物）を一つ一つの船に乗せる技術。
- **Kubernetes**: 港全体の管制塔。どの船をどのドックに入れるか、船が故障したらどうするか、荷物が増えたら船を増やすか、などを自動で管理します。

## 2. なぜDockerだけでは足りないのか？
`docker-compose` は数個〜十数個のコンテナを単一のサーバーで動かすのには適しています（TASUKIのローカル開発環境など）。
しかし、本番環境で**数百・数千のコンテナ**を**複数のサーバー**にまたがって動かす場合、人間が手動で管理するのは不可能です。

K8sは以下のことを自動でやってくれます：
- **自己修復 (Self-healing)**: コンテナがクラッシュしたら、勝手に再起動してくれます。
- **スケーリング (Scaling)**: アクセスが増えたらコンテナを増やし、減ったら減らします。
- **ロードバランシング**: 通信を複数のコンテナにいい感じに振り分けます。

## 3. クラスター (Cluster)
K8sのシステム全体のことです。
- **Master Node (Control Plane)**: 指示を出す「脳」。
- **Worker Node**: 実際にコンテナが動く「手足」。サーバー（マシン）そのもの。

## 4. 基本的な考え方：宣言的設定 (Declarative Configuration)
「今すぐコンテナを3つ起動しろ！」と命令するのではなく、
**「Webサーバーのコンテナが常に3つある状態にしておけ」** とあるべき状態（Desired State）をファイルに書きます（マニフェストファイル）。

K8sは、現在の状態を監視し、指定された状態と違っていれば（例：1つクラッシュして2つになった）、自動で修復（1つ追加）しようとします。
