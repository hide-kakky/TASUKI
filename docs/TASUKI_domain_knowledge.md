# TASUKI ドメイン知識体系

本ドキュメントは、飲食店業界の知識とTASUKIマニュアル作成のベストプラクティスをまとめたものです。

---

## 1. 業務カテゴリとマニュアル分類

### 1.1 カテゴリ定義

TASUKIでは、以下の5つのカテゴリでマニュアルを分類します:

| カテゴリ | 対象業務 | 代表例 |
|---------|---------|--------|
| **ホール** | 接客、配膳、レジ | テーブルセッティング、オーダー受付、会計処理 |
| **キッチン** | 調理、仕込み、盛り付け | 料理の調理手順、食材の下処理、衛生管理 |
| **清掃** | 店内清掃、食器洗浄 | テーブル清掃、トイレ清掃、床清掃 |
| **安全衛生** | 食品衛生、安全管理 | 手洗い、食材保管温度、火器の取り扱い |
| **その他** | 開店/閉店、事務作業 | レジ締め、予約管理、在庫管理 |

### 1.2 カテゴリ選択のガイドライン

- **明確な主業務で判断**: 例えば「テーブル清掃」は、配膳時なら「ホール」、閉店作業なら「清掃」
- **安全衛生は優先**: 手洗いなど衛生管理が主目的なら「安全衛生」
- **迷ったら「その他」**: 複数のカテゴリに該当する場合

## 3. マニュアル作成のベストプラクティス

### 3.1 良いマニュアルの5原則

1. **具体的**: 「丁寧に」ではなく「両手で持つ」
   - ❌ 悪い例: 「料理を丁寧に運ぶ」
   - ✅ 良い例: 「料理の下にトレイを持ち、両手で運ぶ」

2. **簡潔**: 1手順1アクション
   - ❌ 悪い例: 「テーブルを拭いてから食器を並べて、ナプキンを折って置く」
   - ✅ 良い例:
     - Step 1: テーブルを拭く
     - Step 2: 食器を並べる
     - Step 3: ナプキンを折って置く

3. **理由を示す**: なぜその手順が必要かを説明
   - ✅ 良い例: 「手洗いは30秒以上行う（菌を完全に落とすため）」

4. **安全第一**: 危険な作業は必ず注意喚起
   - ✅ 良い例: 「⚠️ 包丁の刃は必ず下向きで洗う（怪我防止）」

5. **視覚的**: 動画や写真があると理解しやすい
   - TASUKIでは動画が自動で付与されます

### 3.2 避けるべき表現

| 避けるべき | 理由 | 代わりに |
|-----------|------|---------|
| 「適量」「少々」 | 曖昧すぎる | 「大さじ1」「ひとつまみ（小さじ1/4）」 |
| 「慣れたら早く」 | 品質が不安定 | 「3分以内を目標に」 |
| 「気をつけて」 | 何に注意かわからない | 「滑りやすいので片手は壁を支える」 |
| 「前の人に聞いて」 | マニュアルの意味がない | 具体的な手順を記載 |
| 専門用語のみ | 新人が理解できない | 「ロマネスコ（カリフラワーの一種）」 |

### 3.3 多言語マニュアルのコツ

1. **シンプルな日本語を使う**: 「ご使用になる」→「使う」
2. **主語を明確に**: 「料理をお客様に提供する」（誰が何をするか明確）
3. **カタカナ語は注釈を**: 「オーダー（注文）」
4. **数字を使う**: 「少しの間」→「3分間」
5. **否定形は避ける**: 「〜してはいけない」→「〜する」

## 4. Gemini プロンプト最適化 Tips

### 4.1 良い動画の撮り方（スタッフ向けガイド）

- **手元をしっかり映す**: 顔は写さなくてOK
- **照明を確保**: 暗いとAIが認識できない
- **ゆっくり動く**: 早すぎるとAIが追えない
- **音声で説明**: 「今から〜します」と声に出す
- **完結させる**: 最初から最後まで一連の流れを録画

### 4.2 AI生成結果の品質向上

**プロンプトに追加すると良い情報**:
```markdown
業務カテゴリ: キッチン
作業レベル: 初級（新人向け）
所要時間: 約5分
注意事項: 火傷に注意
```

**AI が苦手なこと**:
- 細かい温度や時間（動画だけでは判断困難）
- 暗黙の知識（「常識的に」は通じない）
- 店舗固有のルール（「うちの店では」は学習していない）

**対策**:
- Manager が確認時に補足を追加
- manual_edits で修正内容を記録（将来のプロンプト改善に利用）

### 4.3 カテゴリ分類の精度向上

Gemini は以下の情報からカテゴリを判断します:
- **動画の内容**: 作業場所（厨房/ホール/洗い場）
- **音声**: 「お客様に」→ホール、「火を使って」→キッチン
- **動作**: 包丁を使う→キッチン、食器を運ぶ→ホール

**誤分類が多い例**:
- 「テーブルセッティング」→「清掃」と誤認されがち
  - 対策: 音声で「お客様をお迎えする準備」と説明

## 5. マニュアル承認時のチェックポイント

Managerがdraftを承認する際にチェックすべき項目:

### 5.1 内容の正確性
- [ ] 手順が実際の作業と一致しているか
- [ ] 時間や温度が正しいか
- [ ] 危険な作業に注意書きがあるか

### 5.2 わかりやすさ
- [ ] 新人でも理解できる表現か
- [ ] 専門用語に説明があるか
- [ ] 手順が論理的か

### 5.3 多言語対応
- [ ] 外国人スタッフが理解できるか
- [ ] カタカナ語に注釈があるか
- [ ] 数字で具体的に書かれているか

### 5.4 カテゴリの適切性
- [ ] カテゴリ分類が正しいか
- [ ] 検索しやすいタイトルか

## 6. 実践例: カテゴリ別マニュアルテンプレート

### 6.1 ホール業務の例

**タイトル**: オーダーの取り方（基本）

**サマリ** (3行):
```
お客様からオーダーを伺い、ハンディに入力する基本手順です。
笑顔であいさつし、メニューを復唱して確認することが重要です。
ドリンクと料理を分けて聞くとスムーズです。
```

**手順**:
1. テーブルに近づき「お決まりですか？」と声をかける
2. メモまたはハンディを開く
3. まずドリンクを聞く「お飲み物はいかがですか？」
4. 次に料理を聞く「お料理はお決まりですか？」
5. 復唱して確認「〜と〜でよろしいですか？」
6. ハンディに入力し、オーダーを確定

**Tips**:
- 笑顔とアイコンタクトが大切
- メニュー番号で確認すると間違いが減る
- アレルギーがないか必ず確認
